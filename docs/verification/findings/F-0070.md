# F-0070: performance.md still references Java 21 SDK and `./gradlew` despite V8 DRIFT-4 claim

## Severity
P2 (Medium)

## Phase
V8 (re-audit of 2026-04-14 reconciliation)

## Component
docs/performance.md:5

## Invariant or target violated
Doc-vs-code reconciliation. Contradicts ADR-0021 (Maven) and ADR-0022 (Java 25). The V8 pass (2026-04-14) final-report.md §Phase V8 claims "DRIFT-4: performance.md Java version/benchmark config ... Updated to match actual (Java 25, Fork=2)" was resolved.

## Reproducer
1. Open `docs/performance.md` line 5.
2. Observe: `> Run with: \`JAVA_HOME=/home/ubuntu/.sdkman/candidates/java/21.0.10-amzn ./gradlew :configd-testkit:test --tests "io.configd.bench.*"\``
3. Check `/home/ubuntu/Programming/Configd/`: `gradlew` does not exist; `mvnw` and `pom.xml` are the build system.
4. Check `pom.xml`: `<maven.compiler.release>25</maven.compiler.release>` and `<release>25</release>`.
5. Check ADR-0022: "Target Java 25 as the runtime."

## Evidence
```
$ ls /home/ubuntu/Programming/Configd/gradlew
ls: cannot access '.../gradlew': No such file or directory
$ grep -n release /home/ubuntu/Programming/Configd/pom.xml
28:        <maven.compiler.release>25</maven.compiler.release>
101:                        <release>25</release>
```

The V8 report says it was fixed, but only lines 100–102 of performance.md (the JMH methodology row) were actually updated. Line 5's run command and line 339–340's "Running Benchmarks" shell snippet still cite Java 21 and Gradle.

## Root cause
Partial fix during Phase V8. The inline benchmark config block was updated to Java 25/Fork=2/Warmup=5i×1s, but the shell command blocks at the top of the doc and in §10 were overlooked.

## Proposed fix
Replace `JAVA_HOME=.../21.0.10-amzn ./gradlew :configd-testkit:test ...` with the Maven equivalent:
```
JAVA_HOME=<java-25> ./mvnw -pl configd-testkit test -Dtest='io.configd.bench.*'
```
Apply at both line 5 and the `### Running Benchmarks` code block around line 338–342.

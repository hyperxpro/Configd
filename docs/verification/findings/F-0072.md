# F-0072: ADR-0014 (Accepted) references Netty, gRPC-java, and Spring Boot as dependencies that do not exist in pom.xml

## Severity
P1 (High)

## Phase
V8 (re-audit of 2026-04-14 reconciliation)

## Component
docs/decisions/adr-0014-zgc-shenandoah-gc-strategy.md (Status: Accepted)

## Invariant or target violated
Doc-vs-code reconciliation. ADR-0014 names specific libraries:
- "JCTools: Lock-free concurrent queues used by **Netty** and Reactor"
- Rejected alternative "No JVM": "sacrifices the JVM ecosystem (**Netty, gRPC-java**, virtual threads, JCTools, Agrona)"
- "Why not GraalVM Native Image? ... **Spring Boot (used for control plane API)** has GraalVM support but with complexity"

None of Netty, gRPC-java, or Spring Boot are in `pom.xml`. The actual dependency set is Agrona, JCTools, JUnit, JMH, HdrHistogram, Micrometer, jqwik.

## Reproducer
```
$ grep -E "netty|grpc|spring" /home/ubuntu/Programming/Configd/pom.xml
(no matches)
```

## Evidence
The "Why not GraalVM" paragraph argues against Native Image partly because "Virtual threads (critical for 10K+ connection handling per ADR-0009) have incomplete native image support." This reasoning is fine, but the explicit claim that "Spring Boot is used for the control plane API" is false — `HttpApiServer.java` uses `com.sun.net.httpserver.HttpServer`.

Additionally: ADR-0014 says "JCTools: Lock-free MPSC/SPSC queues on hand-off points between threads." Performance.md §2 says "JCTools SPSC/MPSC queues — lock-free, wait-free (**planned, not yet integrated**)." The JCTools dependency is present in pom.xml but the hand-off integration is aspirational per performance.md. The ADR should note this.

## Root cause
Same as F-0071: Phase V8 touched ADR-0010 and ADR-0016 only, leaving ADR-0009 and ADR-0014 with pre-reconciliation claims intact.

## Proposed fix
Add a V8 note to ADR-0014, or strike the specific mentions:
- "Netty and Reactor" → "high-performance Java networking frameworks"
- "Netty, gRPC-java" from the "No JVM" rejected alternative → "…the JVM ecosystem (virtual threads, JCTools, Agrona)"
- "Spring Boot (used for control plane API)" → "the control plane HTTP API (built on the JDK `com.sun.net.httpserver` module)"
- Add a note that JCTools queue integration is planned per performance.md §2.

# F-0055: /metrics Endpoint Is Unauthenticated Even When Auth Token Is Configured

**Severity:** P3 (Low)
**Phase:** V8 — Security Audit
**Status:** Closed — fix: HttpApiServer.MetricsHandler now requires bearer-token auth when AuthInterceptor is configured, returning 401 + WWW-Authenticate; /health/live and /health/ready remain public for probes; test: io.configd.server.ConfigdServerTest::find0055_metricsRequiresAuthWhenAuthConfigured

## Summary

Scope item 4 requires that every HTTP endpoint require the auth token
when one is configured. `HttpApiServer` wires three endpoints that
bypass `AuthInterceptor` unconditionally:

- `GET /health/live`
- `GET /health/ready`
- `GET /metrics`

Health probes bypassing auth is intentional and conventional. `/metrics`
is different: Prometheus exposition can leak sensitive operational
information (leader identity, follower lag, pending proposals, last
applied index, config key cardinality, rate-limit hit counts) and is
generally expected to be protected in production.

## Location

- `configd-server/src/main/java/io/configd/server/HttpApiServer.java:74-78`

```java
server.createContext("/health/live", new LivenessHandler(healthService));
server.createContext("/health/ready", new ReadinessHandler(healthService));
server.createContext("/metrics", new MetricsHandler(prometheusExporter));
server.createContext("/v1/config/", new ConfigHandler(
        configStore, writeService, readService, authInterceptor, aclService));
```

Only `ConfigHandler` receives the `authInterceptor` / `aclService`.
`MetricsHandler` has no auth path.

## Impact

- Any unauthenticated client that can reach the API port can scrape
  `/metrics`. Combined with a plaintext HTTP API (when TLS is also
  off — see residual risk in the report), this is a fully open
  information-disclosure channel.

## Suggested Fix

- Thread `AuthInterceptor` into `MetricsHandler` and reject unauthenticated
  calls when auth is configured, OR
- Bind metrics to a separate, loopback-only port and document it as the
  expected scrape topology.

Also worth considering: require a dedicated `metrics:read` ACL so a
scraper doesn't need `admin`.

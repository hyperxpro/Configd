# F-0074: Performance.md §10 benchmark row "RaftCommitBenchmark ~2000-5000 ops/s per group" contradicts measured 815K/467K ops/s

## Severity
P3 (Low — internal contradiction within the same document)

## Phase
V8 (performance claim consistency audit)

## Component
docs/performance.md lines 322–335 (JMH Benchmark Suite table) vs lines 144–148 (RaftCommitBenchmark measured numbers) vs line 92–93 (Tail Latency Results row).

## Invariant or target violated
Intra-document consistency. Performance.md §3 reports:
- "RaftCommitBenchmark.proposeAndCommit 3 thrpt 3 **0.815 ± 0.851 ops/μs (815K ops/s)**"
- Tail Latency Targets row: "Raft commit (3-node simulated) ... **1.23 μs/op (815K ops/s)** BASELINE"

Performance.md §10 "JMH Benchmark Suite" summary table claims:
- "RaftCommitBenchmark | Single Raft group commit throughput in simulation | **~2000-5000 ops/s per group**"

815,000 vs 2,000–5,000 is a ~3 order-of-magnitude internal contradiction.

## Reproducer
Open `docs/performance.md`; compare line 92 / lines 144–148 with the §10 table at lines 322–335.

## Evidence
Lines 145–147:
```
RaftCommitBenchmark.proposeAndCommit  3  thrpt    3   0.815 ± 0.851  ops/μs  (815K ops/s)
RaftCommitBenchmark.proposeAndCommit  5  thrpt    3   0.467 ± 1.133  ops/μs  (467K ops/s)
```
vs Line 331:
```
| `RaftCommitBenchmark` | Single Raft group commit throughput in simulation | ~2000-5000 ops/s per group |
```

## Root cause
Likely a stale pre-reconciliation figure. The §10 table also reports `VersionedStoreReadBenchmark` as "48 B/op (ReadResult alloc)" but `-prof gc` was explicitly called out in final-report.md §V6 as *not configured*, so the allocation figure is an annotation claim, not a measurement.

## Proposed fix
Update the §10 row to match the measured numbers cited in §3:
- "RaftCommitBenchmark | Single Raft group commit throughput in simulation (in-memory transport) | 815K ops/s (3 voters), 467K ops/s (5 voters); network-limited in production"
Also flag the "0 B/op / 48 B/op" allocation annotations as unverified without `-prof gc`, matching final-report.md §V6 methodology notes.

# F-V7-01: Auth Token Timing Attack via String.equals()

**Severity:** P2 (Medium)
**Phase:** V7 — Security Audit
**Status:** Fixed

## Summary

Authentication token comparison used `String.equals()`, which short-circuits
on the first non-matching character. An attacker can measure response time
differences to recover the token character-by-character (timing side-channel).

## Location

`configd-server/src/main/java/io/configd/server/ConfigdServer.java:273`

## Before

```java
if (expectedToken.equals(token)) {
    return new AuthInterceptor.AuthResult.Authenticated("root", Set.of("admin"));
}
```

## Fix

Replaced with `java.security.MessageDigest.isEqual()` which performs
constant-time byte comparison:

```java
if (java.security.MessageDigest.isEqual(
        expectedToken.getBytes(StandardCharsets.UTF_8),
        token.getBytes(StandardCharsets.UTF_8))) {
    return new AuthInterceptor.AuthResult.Authenticated("root", Set.of("admin"));
}
```

## Regression Test

Existing ConfigdServerTest auth tests verify the comparison still works correctly.
The timing property is a security guarantee, not functionally testable in unit tests.

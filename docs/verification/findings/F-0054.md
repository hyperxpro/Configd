# F-0054: Write Rate Limit Wired at 10 000/s, Not the 1 000/s Claimed in the Security Envelope

**Severity:** P3 (Low)
**Phase:** V8 — Security Audit
**Status:** Closed — fix: docs/performance.md and ConfigdServer.start agree on 10_000/s sustained with 10_000 burst; effective rate printed at boot ("  Write rate   : 10000/s (burst 10000)"); test: io.configd.server.ConfigdServerTest::find0054_writeRateLimiterAtDocumentedEnvelope

## Summary

The Phase V8 scope states the "default 1 000 writes/sec global" is wired.
The code wires 10 000:

```java
// configd-server/.../ConfigdServer.java:306
RateLimiter rateLimiter = new RateLimiter(clock, 10_000, 10_000);
ConfigWriteService writeService =
        new ConfigWriteService(proposer, null, rateLimiter, () -> raftNode.leaderId());
```

Either the documentation or the default is wrong; either way the
"envelope" in `docs/performance.md` and the security documentation does
not match the running configuration.

## Impact

- Capacity planning documents cannot be trusted.
- A 10× higher default than advertised makes it easier to overwhelm a
  cluster from a single compromised client.

## Suggested Fix

- Pick one number, document it, and make it configurable from `ServerConfig`.
- Add a startup log line that prints the effective rate limit.
- Add a test that asserts `rateLimiter.ratePerSecond()` matches the
  documented default.

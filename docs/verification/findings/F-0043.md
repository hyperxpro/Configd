# F-0043: configd-testkit has no JMH uber-jar shade configuration

## Severity
P3

## Phase
V-perf

## Component
configd-testkit/pom.xml

## Invariant or target violated
docs/performance.md references `java -jar configd-testkit/target/benchmarks.jar ...` style invocation (implicit — the standard JMH workflow).

## Reproducer
`./mvnw -q -pl configd-testkit -am -DskipTests package` produces only `configd-testkit-0.1.0-SNAPSHOT.jar` (thin jar with `io.configd.bench.*` classes) and no `benchmarks.jar` / shaded uber-jar. Running benchmarks requires manual classpath construction via `mvn dependency:build-classpath`.

## Evidence
pom.xml contains only `maven-compiler-plugin` with the JMH annotation processor. There is no `maven-shade-plugin` stanza bundling `jmh-core` + module classes into an executable jar with `org.openjdk.jmh.Main` as Main-Class.

Also: benchmarks live under `configd-testkit/src/test/java/io/configd/bench/...`, not `io/configd/testkit/jmh/...` as the verification prompt states. Minor docs drift.

## Proposed fix
Add maven-shade-plugin bound to `package` with:
- `<finalName>benchmarks</finalName>`
- `<transformers><ManifestResourceTransformer><mainClass>org.openjdk.jmh.Main</mainClass>`
- Exclude `META-INF/*.SF` etc.

So CI / docs can reproducibly run `java -jar configd-testkit/target/benchmarks.jar <regex>`.

## Reporter / Reproducer
- Reporter: performance-verifier

## Status
Closed — fix: moved benchmarks from `configd-testkit/src/test/java/io/configd/bench/*` to `configd-testkit/src/main/java/io/configd/bench/*` (so they are compiled for packaging, with JMH dependencies marked `<optional>true</optional>`), and added `maven-shade-plugin` bound to `package` in `configd-testkit/pom.xml`: `finalName=benchmarks`, `ManifestResourceTransformer` with `mainClass=org.openjdk.jmh.Main`, `ServicesResourceTransformer`, `AppendingTransformer` for `META-INF/BenchmarkList` + `META-INF/CompilerHints`, and standard signature/module-info excludes. Reproducibly: `./mvnw -q -pl configd-testkit -am -DskipTests package && java -jar configd-testkit/target/benchmarks.jar <regex>`. Stray `--enable-preview` in `@Fork(jvmArgsAppend=...)` was stripped so the shaded jar runs on plain JDK 25. test: verified with `java -jar configd-testkit/target/benchmarks.jar -l` listing all six benchmarks, and with `java -jar … io.configd.bench.HybridClockBenchmark -f 1 -wi 1 -i 1` end-to-end.

# FIND-0005: Raft tick loop silently dies on uncaught exception

## Severity
Critical

## Discovered by
blue-code-audit-remaining (FIND-C49)

## Confirmed by
Main auditor (ConfigdServer.java:218 uses scheduleAtFixedRate)

## Location
ConfigdServer.java:218

## Description
The Raft tick loop runs inside `ScheduledExecutorService.scheduleAtFixedRate()`. Per the `ScheduledExecutorService` contract, if the task throws any uncaught exception, the executor silently cancels all future executions. The consensus mechanism stops permanently with no heartbeats, no election timeouts, no AppendEntries — the node becomes a zombie that appears alive but does not participate in consensus.

## Impact
Any runtime exception in the tick path (NPE, ArrayIndexOutOfBoundsException, ClassCastException) permanently kills consensus on this node. No alert, no metric, no recovery. If this is the leader, the cluster must wait for election timeout to failover.

## Fix
Wrap the tick body in try-catch that catches Throwable, logs at CRITICAL, and continues.

## Status
Fixed — tick loop wrapped in try-catch(Throwable) with CRITICAL log and continue. Regression test added.

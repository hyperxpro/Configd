# FIND-0007: Authentication disabled by default — unauthenticated production deployments possible

## Severity
Critical

## Discovered by
blue-code-audit-remaining (FIND-C47)

## Confirmed by
Main auditor (ConfigdServer.java:156 — auth-token is optional)

## Location
ConfigdServer.java:156, ServerConfig.java

## Description
When `--auth-token` is omitted (it's an optional parameter), all write/delete/admin endpoints are fully unauthenticated. A forgotten flag means a wide-open production deployment where any network-reachable client can modify the global config store.

## Impact
Unauthorized writes to the config store. Complete system compromise.

## Fix
Either require `--auth-token` (make it mandatory), or fail-open with a loud warning at startup and refuse writes without auth in production mode.

## Status
Mitigated — loud WARNING banner printed to stderr at startup when --auth-token is not set. Full fix (making it mandatory) deferred to avoid breaking existing deployments.
